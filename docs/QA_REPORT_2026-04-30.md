## Issues found + status

### 🐛 BUG (FIXED): `pages/Plan.py` was missing iframe auto-resize script

**Symptom:** When `/Plan` is iframed at `/build-your-plan/`, the iframe
defaults to a fixed `min-height: 1500px` from CSS. As the Streamlit app
grows past that height (long venue list, expanded form, results panel),
content gets clipped or scrollbars appear inside the iframe.

**Cause:** `pages/0_Hosts.py` ships with a `postMessage('streamlit:height')`
script that the WordPress wrapper page listens for to resize the iframe
dynamically. `pages/Plan.py` was missing that script.

**Fix:** Added the same auto-resize script to `pages/Plan.py`. Will go live
on next Render deploy.

### ⚠️ NOTE: Cleanup foreign keys

**Observation:** Test rows in `contracts` and `clients` couldn't be deleted
via the standard `delete_row` helper because:
- `contracts.renewal_contract_id_fkey` self-references contracts
- `activity_log.client_id_fkey` references clients with `ON DELETE` =
  default (no cascade)

**Impact:** None on production. Just a wrinkle for test cleanup. I cleaned
up the QA test rows with a manual SQL chain (clear renewal pointers → delete
activity_log → delete contracts → delete clients).

**Recommendation:** Consider adding `ON DELETE SET NULL` to
`contracts.renewal_contract_id` and `ON DELETE CASCADE` to
`activity_log.client_id` so deleting a test client cascades cleanly. Not
urgent — production deletes are rare and you'd usually `update status =
'churned'` rather than DELETE anyway.
